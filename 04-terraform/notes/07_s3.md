# 07 S3

このメモでは、TerraformでRails Active Storage用S3 Bucketを作成した内容を整理する。

AWS CLI編では、主に以下に対応する。

```text
11_s3_setup.sh
```

SES受信用S3 Bucketは、SES設定と一緒に後続で扱う。

## この段階のゴール

Railsアプリケーションの投稿画像などを保存するS3 BucketをTerraformで作成する。

作成するTerraform resource:

```text
aws_s3_bucket.upload
aws_s3_bucket_ownership_controls.upload
aws_s3_bucket_public_access_block.upload
aws_s3_bucket_server_side_encryption_configuration.upload
aws_s3_bucket_versioning.upload
```

S3追加後の `terraform plan` は以下。

```text
Plan: 60 to add, 0 to change, 0 to destroy.
```

前回のRDSまでが55リソースだったため、S3関連5リソースが増えた。

```text
55 + 5 = 60
```

## variables.tf

Bucket名は変数化した。

```hcl
variable "upload_bucket_name" {
  description = "S3 bucket name for Rails Active Storage uploads."
  type        = string
  default     = "nobu-terraform-iac-lab-upload"
}
```

S3 Bucket名はグローバルで一意である必要がある。

## S3 Bucket

Rails Active Storage用のBucketを作成する。

```hcl
resource "aws_s3_bucket" "upload" {
  bucket = var.upload_bucket_name

  tags = merge(local.common_tags, {
    Name = var.upload_bucket_name
  })
}
```

## Ownership Controls

ACLに依存しない構成にするため、Object OwnershipをBucket owner enforcedにする。

```hcl
resource "aws_s3_bucket_ownership_controls" "upload" {
  bucket = aws_s3_bucket.upload.id

  rule {
    object_ownership = "BucketOwnerEnforced"
  }
}
```

ポイント:

- ACLを使わずBucket Policy / IAM Roleで制御する
- RailsアプリケーションからはWeb EC2のIAM Role経由でアクセスする

## Public Access Block

Bucket自体は公開しない。

```hcl
resource "aws_s3_bucket_public_access_block" "upload" {
  bucket = aws_s3_bucket.upload.id

  block_public_acls       = true
  block_public_policy     = true
  ignore_public_acls      = true
  restrict_public_buckets = true
}
```

投稿画像の保存先ではあるが、BucketをPublicにしない。

アプリケーション側で必要に応じて署名付きURLやRails経由の配信を使う。

## Server Side Encryption

SSE-S3でサーバーサイド暗号化を有効にする。

```hcl
resource "aws_s3_bucket_server_side_encryption_configuration" "upload" {
  bucket = aws_s3_bucket.upload.id

  rule {
    apply_server_side_encryption_by_default {
      sse_algorithm = "AES256"
    }
  }
}
```

## Versioning

学習環境ではコストを抑えるため、VersioningはSuspendedにする。

```hcl
resource "aws_s3_bucket_versioning" "upload" {
  bucket = aws_s3_bucket.upload.id

  versioning_configuration {
    status = "Suspended"
  }
}
```

本番環境では誤削除対策としてVersioning有効化を検討する。

## IAM Roleとの関係

Web EC2には学習用に `AmazonS3FullAccess` を付与している。

```text
aws_iam_role_policy_attachment.web_s3
```

そのため、このBucketへRails Active Storageから書き込みできる。

後続で最小権限化する場合は、対象Bucketだけに絞ったIAM Policyへ変更する。

## outputs.tf

Bucket名とARNをoutputした。

```hcl
output "upload_bucket_name" {
  description = "S3 bucket name for Rails Active Storage uploads."
  value       = aws_s3_bucket.upload.bucket
}
```

```hcl
output "upload_bucket_arn" {
  description = "ARN of the S3 bucket for Rails Active Storage uploads."
  value       = aws_s3_bucket.upload.arn
}
```

## 確認観点

`terraform plan` で見るポイント:

- Bucket名が想定どおりである
- Public Access Blockが有効である
- Ownership Controlsが設定される
- SSE-S3暗号化が有効である
- Versioningは学習環境ではSuspendedである

## コスト注意

S3は保存容量とリクエスト数に応じて課金される。

このラボでは画像アップロード確認程度なので小さいが、不要なオブジェクトを残し続けないようにする。
